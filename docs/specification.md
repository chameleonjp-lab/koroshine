# コロシャイン 仕様書

## ゲーム概要

- タイトル: コロシャイン
- 説明: くるくる回して落ち続けろ。🌰を集めて高得点を狙え。
- 公開予定URL: https://chameleonjp.codeberg.page/koroshine/
- 形式: スマホ向けブラウザゲーム。`index.html` 単体公開可能構成。
- 参考: 既存ゲームの仕組みのみ参考にし、コピーはしない。

## ゲームルール

- 太陽（球）は**重力で落下し、床に当たると跳ね返る**（物理演算: 重力加速・速度積分・反発）。
- プレイヤーは左右ドラッグのみで塔を回転させる。
- 各階の床は中実のスライス（`SEG=8` 分割）で構成され、各スライスは「穴（すき間）」「安全帯」「危険帯」のいずれか。正面が穴なら通過、安全帯なら跳ね返り、危険帯なら終了。
- 穴を通すと下階へ落下する。跳ねずに連続で落ちるとコンボになる。
- **3連続落下でファイアボール化**し、一定時間（`FIRE_TIME`）すべてのスライス（危険帯含む）を破壊して貫通落下する。
- スライスの色自体が状態を表す（安全帯＝ゾーン配色 / 危険帯＝高コントラストな対比色 / 穴＝空白）。現在階の接触スライスはわずかに明るく強調する。判定値はこの表示と一致する。
- 通常時に危険帯へ乗るとゲーム終了。ファイアボール中は破壊して突破する。
- 🌰を取得するとスコアを得る。
- 🌰取得演出では、取得対象が `🌰` から `☀️` に変化し、`+50` を表示する。

## 物理演算

- 重力加速度 `GRAV` で鉛直速度 `bvy` を積分し、位置 `by` を更新する。`bvy` は上限 `MAXV` でクランプする。
- 安全帯に着地した場合は反発初速 `BOUNCEV` で上方へ跳ね返し、コンボをリセットする。
- 穴を正面に合わせた場合は反発せず落下を継続し、`+100`・コンボ・一気落下判定を行う。
- ファイアボール中は落下速度を `FIREV` 以上に保ち、安全帯・危険帯のスライスを穴へ変えて（破壊して）通過する。`FIRE_TIME` 経過で通常状態へ戻る。
- フレーム変動やiOSの低フレームでも一定挙動とするため、固定タイムステップ `1/120` 秒＋アキュムレータで物理を進め、トンネリングを防ぐ。
- 当たり判定は正面角度 `front = normAngle(π/2 - 回転角)` のスライス種別（穴/危険/安全）で行い、描画の正面と完全一致させる。
- v1.1 以降: 回転は角速度（慣性）で操作し、接触は球の幅（中心・左右端の3点）で判定する。詳細は「v1.1 操作感・接触感・立体感 改善」を参照。

## 勝利条件

- 30階到達で勝利。
- 30階到達とは、29階の穴を通過して表示上の現在階層が30階になった時点を指す。
- 30階到達時に `+3000点` を加算する。

## 敗北条件

- 危険床（危険帯）への接触で敗北。ただしファイアボール中は破壊して突破する。
- 危険床接触時は即座に結果画面へ移らず、約0.38秒だけ失敗箇所を表示してから結果画面へ進む。待機中はスコア加算、当たり判定、ランキング通信を行わない。
- リタイア操作で任意終了。誤操作防止のため、リタイアは確認ダイアログでの了承後に終了する。
- リタイア時も結果画面へ進み、その時点のスコアをランキング送信する。ただし30階到達扱いにはしない。

## スコア

| 条件 | 加点 |
| --- | ---: |
| 1階突破 | +100 |
| 🌰取得 | +50 |
| 2連続落下コンボ | +50 |
| 3連続落下コンボ | +100 |
| 4連続落下コンボ | +150 |
| 5連続以降 | 1段ごとに +50 ずつ増加 |
| 3階以上一気落下（ファイアボール化） | +300 |
| ファイアボール中の各階通過 | +50 |
| 30階到達 | +3000 |

- 3階以上一気落下ボーナスは、850ms以内の穴通過が3回連続したときに `+300` し、ファイアボール（`FIRE_TIME=0.48` 秒）を発動する。
- ファイアボール中は各階通過ごとに `+50`（床破壊・穴通過いずれも同額）。1発の発動でおおむね5〜6階を貫通する。
- プレイ中は2連続以上でコンボ表示を行い、3連続以上では一気落下チャンスを表示する。
- 一気落下予告として、`dropBurst` が1なら `連続落下 1/3`、2なら `あと1回で一気落下` を表示し、850ms以上空いたら消す。
- 連続状態が続く場合も、一気落下ボーナスは3回ごとに1回まで発生する。
- 穴通過の間隔が850ms以上空いたら、一気落下カウントをリセットする。
- ランキングは高いスコアが上位。並び順は `desc`。
- `RESULT` 中はスコア加算を停止する。

## ランキング

- 本番Supabaseランキング連携を使用する。
- Supabase URL: `https://mlpnjgezrnhdxsxolyzj.supabase.co`
- Publishable key: 公開HTMLに配置可能な `sb_publishable_...` キーを使用する。
- secret key / service_role key は使用禁止。
- `public.scores` は使用禁止。
- 利用対象は `public.games` / `public.game_scores` / 既存RPC。
- スコア送信RPCは `submit_score`。
- ランキング取得RPCは `get_best_score_ranking`。
- プレイ統計RPCは必要に応じて `get_game_play_stats`。
- `game_slug` は `koroshine` 固定。
- 送信値は `p_display_name` / `p_game_slug` / `p_score` / `p_client_version`。
- スコア種別はベストスコア型で、高い方が良い。
- `score_order`: `desc`、`score_unit`: `点`、`score_scale`: `1`、`score_decimals`: `0`、`top_ranking_type`: `best`。
- ゲーム終了時に自動送信する。クリア、危険床接触、リタイアのいずれも対象。
- 送信は1プレイにつき1回のみ。
- ランキング取得も結果画面表示時に1回のみ。
- 結果画面では結果を先に表示し、その後に送信中、成功または失敗、ランキング取得結果を表示する。
- 送信失敗時も結果表示を継続し、画面には `ランキング送信に失敗しました。スコアはこの画面で確認できます。` と短く表示する。
- 取得失敗時は `ランキング取得に失敗しました。時間をおいて再度お試しください。` と短く表示する。
- 詳細エラーは画面に長く表示せず、開発確認用に `console.warn` へ出す。
- ローカルランキングを本番ランキング欄の代替として混同表示しない。
- 結果画面にランキング登録ボタンは置かない。

## 状態管理

必須状態は以下。

- `HOME`
- `RULE`
- `NAME`
- `READY`
- `PLAYING`
- `RESULT`
- `ERROR`

状態別の制約:

- `PLAYING` のみゲーム更新・当たり判定・スコア加算を行う。
- `RESULT` 中はスコア加算、当たり判定、ゲーム更新を停止する。
- `ERROR` は致命的な初期化失敗などの表示に使う。

## v1.1 操作感・接触感・立体感 改善

ゲーム内容（スコア式・ランキング・画面遷移・30階クリア・ファイアボール・単一HTML・外部依存ゼロ）は変えず、操作感と表現のみ改善した。本v1.1は Codex 実装（PR #10）と本実装を統合したもの。慣性・通過/失敗演出は Codex 実装を採用し、通過判定（依頼書準拠の球幅）と疑似3D奥行き（Phase 5）を本実装で上乗せしている。

### Phase 1: 回転操作に「重さ」（慣性）— Codex実装を採用
- 追加定数: `MAX_ANG_V=8.5`（最大角速度・rad/秒）/ `ANG_DRAG=15`（減衰係数）。
- 追加状態: `game.angularVelocity`（rad/秒）、`game.input.lastT`。
- 変更関数: `pointermove`（指の移動量と経過時間から目標角速度 `clamp(dx*SENS/dt, ±MAX_ANG_V)` を求め、`av=av*0.45+target*0.55` で追従）、`pointerdown`（掴んだ瞬間 `av*=0.35` で減衰）、`stepAnims(dt)`（`angle+=av*dt`、`av*=max(0,1-ANG_DRAG*dt)` で減速、`|av|<0.01` で停止）。秒基準のため fps 差に強い。
- 挙動: 指の速度に追従して回り、離すと慣性で少し回って止まる。
- 変更しない: `SENS`、スコア計算。

### Phase 2: 接触判定に「球の幅」を反映（通過判定は依頼書準拠に調整）
- 追加関数: `collisionAt(p)` → `{front,span,center,left,right,type,result,wi}`。`wedgeIndexAt(a)` は角度引数対応で残す。
- 角度半幅: `span = min(WS*0.44, asin(min(0.95, RBALL/R))*1.15)`。正面の中心・左端・右端のくさびを見る。
- 判定（依頼書準拠）: **中心・左端・右端すべてが穴なら pass**、いずれかが危険帯なら danger、中心が穴でも端が床なら edge（=跳ね）、それ以外は bounce。ファイアボール中は pass 以外を破壊（`smash`）。
- 接続関数: `stepPhysics()` を `collisionAt(p)` 経由に変更。
- `?debug` 時のみ `result/center/left/right/wi` と角度・span を画面下部に表示（`game.debugHit`・`drawHints`）。通常プレイは非表示。
- 挙動: 穴の中央のみ安定通過、端は跳ね、危険帯に少しでも触れたら失敗。見た目と判定のズレを低減。

### Phase 3: 床通過の瞬間の演出 — Codex実装を採用
- 追加状態: `game.passFX = {x,y,life,max,zone,gStart,gW,angle}`。
- 変更関数: `passFloor()`（通過時に `passFX` 起動・`squash=-1` で縦伸び・小粒子）、`stepAnims()`（`passFX.life` 減衰・通過中はカメラ追従を `dt*20` に加速・負の `squash` を戻す）、新規 `drawPassFX()`（穴の縁を発光）、`drawBall()`（`squash<0` で縦に伸びる）。
- 変更しない: スコア・階層進行・クリア判定。

### Phase 4: 危険床の失敗演出強化 — Codex実装を採用
- 追加状態: `game.failHit`（`collisionAt` の結果）。
- 変更関数: `die()`（接触した危険くさび `failHit.wi` を保存、`squash=1.15`）、`drawPlatform()`（`pendingFinish` 中は接触した危険くさびを赤くハイライト＋縁取り）。
- 変更しない: 結果理由「危険床接触」、ランキング送信1回。

### Phase 5: 疑似3Dの奥行き強化（Canvas 2Dのまま）— 本実装
- 変更関数: `drawPlatform()`。各くさびの画面前後 `sin((i+0.5)*WS+phi)` で手前を明るく（白を最大10%）・奥を暗く（黒を最大12%）する薄いオーバーレイを1枚追加。外周の影は既存の側面押し出し（暗色）で表現。
- 変更しない: ゾーン配色設計、危険帯と安全床のコントラスト。

### 検収（iPhone SE 320/375・iOS UA・実ブラウザ自動プレイ）
- 慣性: ドラッグで角速度生成（約2.5 rad/秒）→離すとコースト→停止。
- 球幅判定: 穴の中央=通過（result=pass）/ 穴の端=跳ね返り（result=edge, 同階に留まる）/ 危険帯接触=失敗。
- 通過演出: `passFX` 起動、縁の発光。
- 危険演出: 接触した危険くさびを赤くハイライト、理由「危険床接触」。
- 立体感: 手前/奥の明暗差。危険帯の視認性維持。
- 回帰なし: 30階クリア・ファイア中の危険帯貫通・本番でデバッグ非公開・JSエラー0・送信1/取得1。
