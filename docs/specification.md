# コロシャイン 仕様書

## ゲーム概要

- タイトル: コロシャイン
- 説明: くるくる回して落ち続けろ。🌰を集めて高得点を狙え。
- 公開予定URL: https://chameleonjp.codeberg.page/koroshine/
- 形式: スマホ向けブラウザゲーム。`index.html` 単体公開可能構成。
- 参考: 既存ゲームの仕組みのみ参考にし、コピーはしない。

## ゲームルール

- 球は自動落下する。
- プレイヤーは左右ドラッグのみで塔を回転させる。
- 各階の穴に球が通ると下階へ落下する。
- 危険床に接触するとゲーム終了。
- 🌰を取得するとスコアを得る。
- 🌰取得演出では、取得対象が `🌰` から `☀️` に変化し、`+50` を表示する。

## 勝利条件

- 30階到達で勝利。
- 30階到達時に `+3000点` を加算する。

## 敗北条件

- 危険床への接触で敗北。
- リタイア操作で任意終了。

## スコア

| 条件 | 加点 |
| --- | ---: |
| 1階突破 | +100 |
| 🌰取得 | +50 |
| 2連続落下コンボ | +50 |
| 3連続落下コンボ | +100 |
| 4連続落下コンボ | +150 |
| 5連続以降 | 1段ごとに +50 ずつ増加 |
| 3階以上一気落下 | +300 |
| 30階到達 | +3000 |

- ランキングは高いスコアが上位。並び順は `desc`。
- `RESULT` 中はスコア加算を停止する。

## ランキング

- `public.scores` は使用禁止。
- 利用対象は `public.games` / `public.game_scores` / 既存RPC。
- ゲーム終了時に自動送信する。
- 送信は1プレイにつき1回のみ。
- ランキング取得も結果画面表示時に1回のみ。
- 送信失敗時も結果表示を継続し、エラーメッセージを表示する。
- 結果画面にランキング登録ボタンは置かない。

### 曖昧点

- 既存RPC名、URL、anon key、引数名、戻り値形式が未提示。
- そのため初期実装ではランキング処理を設定可能なRPCラッパーとして実装し、未設定時はローカル表示と失敗メッセージで継続する。

## 状態管理

必須状態は以下。

- `HOME`
- `RULE`
- `NAME`
- `READY`
- `PLAYING`
- `RESULT`
- `ERROR`

状態別の制約:

- `PLAYING` のみゲーム更新・当たり判定・スコア加算を行う。
- `RESULT` 中はスコア加算、当たり判定、ゲーム更新を停止する。
- `ERROR` は致命的な初期化失敗などの表示に使う。

## スマホ対応

- iPhone中心。
- iPhone SE幅に対応する。
- 操作は左右ドラッグのみを優先する。
- 以下を禁止・抑制する。
  - ピンチズーム
  - ダブルタップ拡大
  - 長押し選択
  - テキスト選択
  - 横スクロール

## 必須演出

- 落下時の軽い振動演出。
- コンボ演出。
- 🌰取得時の `🌰 → ☀️` 演出と `+50` 表示。
- 危険床接触演出。
- クリア演出。
- タップやドラッグ時の即時反応。

## 性能予算

- 目標: 60fps。
- 最低: 30fps維持。
- 表示床: 最大8階。
- アイテム: 最大8個。
- エフェクト: 最大60個。
- 描画: 最大100要素程度。
- DOM更新: 最小限。
- ランキング通信: 送信1回、取得1回。
