# コロシャイン 仕様書

## ゲーム概要

- タイトル: コロシャイン
- 説明: くるくる回して落ち続けろ。🌰を集めて高得点を狙え。
- 公開予定URL: https://chameleonjp.codeberg.page/koroshine/
- 形式: スマホ向けブラウザゲーム。`index.html` 単体公開可能構成。
- 参考: 既存ゲームの仕組みのみ参考にし、コピーはしない。

## ゲームルール

- 球は自動落下する。
- プレイヤーは左右ドラッグのみで塔を回転させる。
- 各階の穴に球が通ると下階へ落下する。
- 現在階の穴と球の角度が近いとき、判定を変えずに穴または床を軽く光らせる。通過可能範囲ではさらに明るく表示する。
- 危険床に接触するとゲーム終了。
- 🌰を取得するとスコアを得る。
- 🌰取得演出では、取得対象が `🌰` から `☀️` に変化し、`+50` を表示する。

## 勝利条件

- 30階到達で勝利。
- 30階到達とは、29階の穴を通過して表示上の現在階層が30階になった時点を指す。
- 30階到達時に `+3000点` を加算する。

## 敗北条件

- 危険床への接触で敗北。
- 危険床接触時は即座に結果画面へ移らず、約0.35秒だけ失敗箇所を表示してから結果画面へ進む。待機中はスコア加算、当たり判定、ランキング通信を行わない。
- リタイア操作で任意終了。
- リタイア時も結果画面へ進み、その時点のスコアをランキング送信する。ただし30階到達扱いにはしない。

## スコア

| 条件 | 加点 |
| --- | ---: |
| 1階突破 | +100 |
| 🌰取得 | +50 |
| 2連続落下コンボ | +50 |
| 3連続落下コンボ | +100 |
| 4連続落下コンボ | +150 |
| 5連続以降 | 1段ごとに +50 ずつ増加 |
| 3階以上一気落下 | +300 |
| 30階到達 | +3000 |

- 3階以上一気落下ボーナスは、850ms以内の穴通過が3回連続したときに `+300` する。
- プレイ中は2連続以上でコンボ表示を行い、3連続以上では一気落下チャンスを表示する。
- 一気落下予告として、`dropBurst` が1なら `連続落下 1/3`、2なら `あと1回で一気落下` を表示し、850ms以上空いたら消す。
- 連続状態が続く場合も、一気落下ボーナスは3回ごとに1回まで発生する。
- 穴通過の間隔が850ms以上空いたら、一気落下カウントをリセットする。
- ランキングは高いスコアが上位。並び順は `desc`。
- `RESULT` 中はスコア加算を停止する。

## ランキング

- 本番Supabaseランキング連携を使用する。
- Supabase URL: `https://mlpnjgezrnhdxsxolyzj.supabase.co`
- Publishable key: 公開HTMLに配置可能な `sb_publishable_...` キーを使用する。
- secret key / service_role key は使用禁止。
- `public.scores` は使用禁止。
- 利用対象は `public.games` / `public.game_scores` / 既存RPC。
- スコア送信RPCは `submit_score`。
- ランキング取得RPCは `get_best_score_ranking`。
- プレイ統計RPCは必要に応じて `get_game_play_stats`。
- `game_slug` は `koroshine` 固定。
- 送信値は `p_display_name` / `p_game_slug` / `p_score` / `p_client_version`。
- スコア種別はベストスコア型で、高い方が良い。
- `score_order`: `desc`、`score_unit`: `点`、`score_scale`: `1`、`score_decimals`: `0`、`top_ranking_type`: `best`。
- ゲーム終了時に自動送信する。クリア、危険床接触、リタイアのいずれも対象。
- 送信は1プレイにつき1回のみ。
- ランキング取得も結果画面表示時に1回のみ。
- 結果画面では結果を先に表示し、その後に送信中、成功または失敗、ランキング取得結果を表示する。
- 送信失敗時も結果表示を継続し、画面には `ランキング送信に失敗しました。スコアはこの画面で確認できます。` と短く表示する。
- 取得失敗時は `ランキング取得に失敗しました。時間をおいて再度お試しください。` と短く表示する。
- 詳細エラーは画面に長く表示せず、開発確認用に `console.warn` へ出す。
- ローカルランキングを本番ランキング欄の代替として混同表示しない。
- 結果画面にランキング登録ボタンは置かない。

## 状態管理

必須状態は以下。

- `HOME`
- `RULE`
- `NAME`
- `READY`
- `PLAYING`
- `RESULT`
- `ERROR`

状態別の制約:

- `PLAYING` のみゲーム更新・当たり判定・スコア加算を行う。
- `RESULT` 中はスコア加算、当たり判定、ゲーム更新を停止する。
- `ERROR` は致命的な初期化失敗などの表示に使う。

## スマホ対応

- iPhone中心。
- iPhone SE幅に対応する。
- 操作は左右ドラッグのみを優先する。
- 以下を禁止・抑制する。
  - ピンチズーム
  - ダブルタップ拡大
  - 長押し選択
  - テキスト選択
  - 横スクロール

## 必須演出

- 落下時の軽い振動演出。
- コンボ演出。
- 🌰取得時の `🌰 → ☀️` 演出と `+50` 表示。
- 危険床接触演出。
- 現在階の穴が近いときの視覚補助。
- コンボ状態表示と一気落下予告。
- 結果画面のプレイ評価コメント。
- クリア演出。
- タップやドラッグ時の即時反応。

## 性能予算

- 目標: 60fps。
- 最低: 30fps維持。
- 表示床: 最大8階。
- アイテム: 最大8個。
- エフェクト: 最大60個。
- 描画: 最大100要素程度。
- DOM更新: 最小限。
- ランキング通信: 送信1回、取得1回。
