## 1. テスト実施方法と結果（テストプレイ）

実機ブラウザバイナリ（`/opt/pw-browsers` の Chromium）をiPhone SE相当でエミュレーション（375×667 / 320×568、dpr2、touch、iOS Safari UA）して自動テストプレイを実施。あわせて `node --check` 構文検証・静的精読を行った。

### 1-1. 旧実装で再現した致命的課題（修正前ベースライン）

| ID | 重大度 | 内容（テストで観測） |
| --- | --- | --- |
| **B1** | 致命 | **球が床で停止し「動かない」。** 床に当たり穴と非整列だと `ball.y=y-8` で静止し、回転で合わせるまで待機。自動テストで現在階が `2/30` のまま4秒間停止。プレイヤーには「落ちない＝壊れている」と映る。 |
| **B2** | 致命 | **描画モデルが理解不能。** 全リングが同半径・同心で縦に重なり「バネ/コイル」状になり、どこが床・穴・現在位置か読み取れない（スクリーンショットで確認）。 |
| **B3** | 高 | **物理演算が存在しない。** 落下は等速 `vy=92`、床送りは `ball.y % 58` のモジュロ擬似スクロール。重力・反発・慣性がなく「リアリティ」がない。要望に真っ向から反する。 |

### 1-2. 刷新後エンジンの検証結果（修正後）

iPhone SE 320/375 双方で自動プレイ：

- Canvas解像度: 600×810 / 710×1082（dpr2で正しく確定）。
- 自由落下: 入力なしで `by` が約108px往復し、`bvy` が正負に振れる＝**重力で落ち、跳ね返る**。
- 進行: 穴を正面に合わせ続けると **30階クリア（スコア29200）**。
- 失敗: 危険帯を正面に合わせると **6階で「危険床接触」終了**。
- 実ドラッグ: 68px ドラッグ＝0.852rad 回転（感度一致）、合わせて床通過を確認＝**入力経路全体が機能**。
- 本番経路（`?debug`なし）でデバッグ用フック非公開、JSエラー0、通信は送信1回＋取得1回、favicon 404なし。

## 3. ロール別 課題抽出と意見

### R1 ゲームデザイナー
- 旧コアは「合わせるまで停止」で待ち時間が長く爽快感ゼロ（B1）。落ち続ける気持ちよさが核のはず。
- 提案: **落下＝デフォルト、跳ね返り＝安全床、穴で一気に落下**という「ヘリックス系」物理ループへ。連続落下が自然にコンボへ繋がる。
- 危険は「赤い帯に乗ると終了」に統一し、毎フロアに穴・安全・危険を配置して常時判断を発生させる。

### R2 物理/描画エンジニア
- B3: 重力加速・速度積分・反発（restitution）を導入。可変フレームでも破綻しないよう**固定タイムステップ（1/120）＋アキュムレータ**でトンネリング防止。
- B2: 同心円をやめ、**遠近楕円リングを縦に積層**（見下ろし `PERSP=0.34`）＋中央支柱で「塔」を明示。当たり判定は「正面セグメント＝穴/危険/安全」の角度判定に単純化（描画と完全一致）。
- B4: `ResizeObserver` で実寸確定、`loop` 内で `H<2` なら再 `resize`、`orientationchange`/`visualViewport` も購読。

### R3 UI/UXデザイナー
- 初見で「どこを狙うか」が分かる必要がある。**正面（接触点）に状態色リング**（緑=穴/赤=危険/白=床）を表示。
- 太陽は絵文字依存をやめ自前描画（放射光の回転・スクワッシュ＆ストレッチ・影・残像）で生き生きと。落下/反発/取得/失敗に粒子と振動でフィードバック。
- 配色は太陽テーマの暖色グラデ、危険のみ高彩度ピンク赤でコントラスト確保。

### R4 モバイル品質エンジニア
- `min-height` は `100vh`→`100dvh` の二段指定でdvh非対応iOSでも床面を確保。Canvasラッパは `min-height:60vh` を保険に。
- 読み込みは外部依存ゼロ（HTML1枚）。デバッグ公開は `?debug` 時のみ。a11y（`aria-live`・Enter送信・シェア多段フォールバック）は維持。
- 既存のランキング/シェア/状態機械の契約は壊さない（回帰防止）。

## 5. 採用項目の実装計画（`index.html`）

1. **物理コア（P-CORE/P-FIXED）**: `by, bvy` を重力 `GRAV=2200` で積分。`bvy` は `MAXV=1600` で上限。落下中に現在ターゲット床 `platforms[floor-1].y` を跨いだら、正面セグメントを判定。
   - 穴: 通過（`+100`、コンボ加点、`floor++`、🌰取得）。
   - 安全: 反発（`bvy=-BOUNCEV`、コンボリセット、スクワッシュ、粒子）。
   - 危険: 0.38秒の失敗表示後に終了。
   - `loop` は `dt` をクランプし、`while(acc>=1/120)` で物理を固定ステップ実行（最大10ステップ）。
2. **描画モデル（P-RENDER/P-AIM/P-JUICE）**: 各床を `ctx.ellipse`（`PERSP=0.34`）の遠近リングとして縦積層。穴以外を一周ストロークし、危険帯を赤で重畳、厚み用に暗色を `THICK` 下へ。中央支柱・階数ラベル・正面状態色リングを描画。太陽は放射光回転＋放射状グラデ＋顔＋スクワッシュ、影・残像・粒子・画面シェイクを追加。カメラは落下に速く・跳ね上がりに緩く追従。
3. **当たり判定**: `front = normAngle(π/2 - angle)`。描画の正面（楕円パラメータ角 `π/2`）と一致するため、見た目と判定が常に同期。
4. **サイズ確定（P-RESIZE）**: `resize()` は実寸→`canvas.width/height=実寸×dpr`、`view.{W,H,R,RBALL}` 更新。`ResizeObserver(wrap)`・`window.resize`・`orientationchange`・`visualViewport.resize` で再計算。`loop` 冒頭で `H<2` なら再 `resize`。CSSは `min-height:100vh→100dvh` 二段、ラッパに `min-height:60vh`。
5. **不変契約（P-SCORE ほか）**: スコア式・コンボ・一気落下・🌰→☀️・30階+3000・状態機械・Supabase `submit_score`/`get_best_score_ranking`・1回送信/取得・シェア多段フォールバック・`aria-live`・Enter送信・SVGファビコンを維持。
6. **テスト用フック**: `?debug` を含むURL時のみ `window.__koro`/`__koroForce` を公開（本番は非公開）。

## 6. 影響範囲・リスク

- 変更は `index.html` のみ（ドキュメントは別途更新）。サイズは約31KB（2MB方針内）。外部リクエスト追加なし。
- ゲーム名・URL・操作系・スコア・状態・ランキング契約は不変＝既存ランキングと互換。

- 危険帯は穴と十分な角度マージンを確保し、必ず通過可能（詰み無し）を保証。
