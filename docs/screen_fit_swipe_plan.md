# コロシャイン 画面フィット・スワイプ速度・誤バック防止 実装計画書（v1.4）

対象: `index.html`（単一HTML・外部依存ゼロ・iPhone優先）
作成日: 2026-06-23
入力: 実機（iPhone）スクショ＋指摘3点。
不変: スコア式・Supabase RPC（送信1/取得1）・状態遷移・30階クリア・ファイアボール・単一HTML。

## 1. 要件（指摘の整理）

- **要件A 物理的に到達不可能な配置**: 深層（例26F）で、直前の穴から真裏（≈180°）に穴があり、落下までに回しきれず詰む。「画面の幅的に（1スワイプの回転量では）物理的に無理」。
- **要件B 画面位置が固定されない／端スワイプで誤ブラウザバック**: プレイ中にページがスクロール/揺れ、左端スワイプでSafariが戻る。
- **要件C スワイプ速度で回転量を可変に**: 速く動かしたら大きく回る（現状は移動量×固定係数の直接マッピングのみ）。

3点は連動する：Cの「速いほど大きく回る」が実装されれば、Aの遠い穴も素早いフリックで届く。Aは生成側でも到達可能に緩和する。

## 2. 役割分担（すべて Opus）

| ロール | 観点 |
| --- | --- |
| R1 ゲームデザイナー | 難易度・公平性・到達可能性（生成リバランス） |
| R2 入力エンジニア | スワイプの速度ゲイン（安定性との両立） |
| R3 モバイルフロント | ビューポート固定・端スワイプ誤バック抑止・全幅化 |
| R4 QA | 実機相当テスト・回帰 |

## 3. ロール別 方針と採否

### R2 スワイプ速度ゲイン（要件C）
- 現状 `angle += dx*SENS`（速度非依存）。→ **`gain = 1 + min(SWIPE_GAIN_MAX, |dx|*SWIPE_GAIN_K)`** を掛け、`angle += dx*SENS*gain`。速い（=1イベントの移動量が大きい）ほど大きく回る。
- 安定性: Gemini指摘の `dt` 割り算は使わず**移動ピクセル量**でゲインを出すため、フレームレートで暴れない。1イベントの回転量は `STEP_CAP` で上限（誤検知の瞬間移動を防止）。
- 離した後の慣性（`lastDragStep`）はゲイン後の実回転量を引き継ぐ。
- 定数: `SWIPE_GAIN_K=0.03, SWIPE_GAIN_MAX=2.2, STEP_CAP=0.9`（実機調整可）。

### R1 生成リバランス（要件A）
- ルール1中盤を「真裏（`SEG/2±1`＝135〜225°）」から**「2〜3ブロックの中程度オフセット・方向もシード」**へ緩和（90〜135°、フリックで届く範囲）。連続で真裏に飛ばさない。
- 危険帯の上限を **4→3** に下げ、増加開始も後ろ倒し（`n>=4:1 / n>=12:2 / n>=22:3`）。深層が一面赤で潰れる問題を緩和し、安全な着地余地を確保。
- 既存の「穴の両隣が両方危険なら片方を安全へ戻す」詰み防止は維持。穴中央は球幅判定で必ず通過可能。

### R3 ビューポート固定・誤バック抑止（要件B）
- プレイ中のみ `<html>` に `playing` クラスを付与し、`position:fixed; inset:0; overflow:hidden; overscroll-behavior:none; touch-action:none` で**ページを完全固定**（アドレスバー伸縮によるズレ・スクロールを排除）。
- ゲーム画面を**全幅化**（プレイ中は左右パディング0）し、左右端まで `touch-action:none` の領域で覆って端スワイプを物理的に拾わせない。
- プレイ中のみ `document` の `touchmove` を `preventDefault`（`passive:false`）し、ラバーバンド/パン/端ジェスチャを抑止。結果画面のスクロールは阻害しない（PLAYING時のみ）。
- 既存の `gesturestart` 抑止は維持。

## 4. 実装手順（`index.html`）

1. 定数追加 `SWIPE_GAIN_K / SWIPE_GAIN_MAX / STEP_CAP`。
2. `pointermove`: 速度ゲイン＋`STEP_CAP`、コースト用に実回転量を積算（`dragDX`→角度積算 `dragStep`）。
3. `makeStage`: 中盤オフセット緩和、危険帯上限を3へ。
4. CSS: `html.playing{…固定…}` と全幅化、`.game` 側の余白調整。
5. `setState`: PLAYING で `documentElement.classList.add('playing')`、その他で除去。
6. `wire`: PLAYING時の `touchmove` preventDefault を追加。

## 5. 検証・完了条件

- スワイプ: 同距離でも**速いほど回転量が大きい**（高速 > 低速）。低速は精密。
- 到達可能性: 全30階、各階の穴中央で通過でき、中盤の必要回転が1〜2フリックで届く範囲。
- 誤バック: プレイ中はページが固定され縦横スクロール・ラバーバンドが出ない（端 `touchmove` がキャンセルされる）。
- 回帰なし: 30階クリア・ファイア6階・危険床赤・本番でデバッグ非公開・JSエラー0・送信1/取得1。
- `node --check` OK。
