参照: Helix Jump（Voodoo）<https://apps.apple.com/jp/app/helix-jump/id1345968745> をイメージ品質の基準とする。
前提: スコア式・Supabase RPC（送信1/取得1）・状態機械・公開URL・左右ドラッグ操作は不変。
方針: 「既存ゲームの仕組みのみ参考にしコピーはしない」原則を守りつつ、**Helix級のビジュアルとゲーム性**へ引き上げる。太陽（コロシャイン）の独自アイデンティティは維持。

## 0. 結論（なぜ今のクオリティでは出せないか）

物理（重力・反発）と「塔」構造は入ったが、Helixの**気持ちよさの核**が欠けている。

1. 床が**中実のスライス円盤ではなく、中空のリング（線）**で、安っぽく奥行きが乏しい。
2. **ファイアボール貫通**（連続落下でボールが燃えて床/危険を破壊して突き抜ける）が無く、コンボの報酬が「数字だけ」。
3. 配色が単調で**図と地のコントラスト**が弱く、危険の警告性も低い。
4. 着地スプラッシュ・ゾーン配色変化など**プレミアム感の演出**が無い。

## 4. 実装計画（`index.html`）

1. **床モデルをスライス化（H-SLAB）**: 各床を `SEG=8` のくさび（wedge）に分割し、各くさびを `gap / safe / danger` で型付け。`gap` は穴（落下）、`safe` は反発、`danger` は危険。
2. **中実スライス描画（H-SLAB）**: くさびを内半径 `r0`〜外半径 `R` の扇形（annular sector）として、遠近 `PERSP` の楕円上に**塗りつぶし**。`THICK` 下に暗色の側面を先に塗って厚みを出す。中央に陰影付き支柱。床は奥（上）→手前（下）の順で描き遮蔽を表現。
3. **当たり判定**: 正面 `front = normAngle(π/2 − 回転角)` → くさび index → 型で判定（描画と完全一致）。
4. **ファイアボール（H-FIRE）**: `dropBurst>=3` で `game.fire=FIRE_TIME`。ファイア中は正面が `safe/danger` でも**破壊**（そのくさびを `gap` 化＝穴、破片粒子、加点）して落下継続。`gap` はそのまま通過。時間切れで通常へ。危険はファイア中のみ無効化。
5. **ゾーン配色（H-PALETTE）**: `zone = floor((n-1)/6)` でパレット（safe/side/danger/背景）を切替。危険は安全色と高コントラストな色に。
6. **ボール刷新（H-BALL）**: 放射状グラデの球＋淡いコロナ＋接地影＋着地スカッシュ＋落下ストレッチ。ファイア中は炎コア＋加算グロー＋火の粒子。回転光線は廃止。
7. **着地スプラッシュ（H-SPLAT）**: 反発時に接地点へ短命のペイント痕＋飛沫粒子。
8. **テンポ調整（H-TEMPO）**: `GRAV/BOUNCEV/SENS/FLOORGAP` を弾む爽快感に調整。固定タイムステップ(1/120)は維持。
9. **不変契約（H-CONTRACT）**: スコア式・コンボ・🌰→☀️・30階+3000・状態機械・`submit_score`/`get_best_score_ranking`・1回送信/取得・シェア・aria-live・Enter送信を維持。`clientVersion` を `koroshine-2.1.0` に更新。

## 5. 影響範囲・リスク

- 変更は `index.html` の `<script>`（描画/物理/ステージ）と一部文言中心。サイズは数十KB（2MB方針内）、外部リクエスト追加なし。
- スライス塗りは可視床 ~7 × 8くさび × 2パス ≈ 112 扇形/フレーム。iPhone SE 60fps想定内（描画ステップ数を抑制）。
- ファイア中の連続破壊は固定時間で打ち切り、詰み/暴走を防止。危険くさびは穴と角度マージンを確保し通過可能性を担保。
