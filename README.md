# コロシャイン

コロシャインは、**重力で落ちて跳ね返る太陽**を直接操作せず、塔を左右ドラッグで回転させて床のすき間（穴）へ通し続ける、物理演算ベースのスマホ向けブラウザゲームです。

公開予定URL: https://chameleonjp.codeberg.page/koroshine/

## ゲーム概要

- タイトル: コロシャイン
- 説明: くるくる回して落ち続けろ。🌰を集めて高得点を狙え。
- 対象: スマホ向けブラウザ、iPhone中心、iPhone SE対応
- 構成: `index.html` 単体公開可能
- その他のゲーム: https://chameleonjp.codeberg.page/chameleonjp_lab/

## 遊び方

1. 名前を入力します。
2. ゲーム開始後、太陽は重力で落下し、床に当たると跳ね返ります。
3. 左右ドラッグで塔を回転させ、床のスライスのすき間（穴）を太陽の正面に合わせます。
4. すき間を通すと下階へ落下します。跳ねずに連続で落ちるほどコンボになります。
5. **3連続で落下するとファイアボール🔥に変身**し、続く6階ぶんのスライス（赤い危険帯も含む）をぶち破って一気に落下します（HUDに残り階数を表示）。
6. 通常時に赤い危険帯に乗ると、短く失敗箇所を表示してからゲーム終了です。
7. 40階到達でクリアです。深い階ほど配色が変わり危険帯が増えますが、**危険帯は全階で赤に統一**されています。

## ユーザープレイ体験の改善

- 現在階の穴が球の真下に近づくと床が光り、通過可能な範囲ではさらに明るくなります。
- 危険床で失敗した場合は、約0.35秒だけ失敗箇所を表示してから結果へ進みます。
- Canvas上にコンボ状態と一気落下予告を表示し、`+300` の一気落下を狙いやすくしています。
- 結果画面には到達階や終了理由に応じたプレイ評価コメントが出ます。
- ランキング通信に失敗しても、長い技術エラーは表示せず、スコア・到達階・シェアはそのまま使えます。

## マップ復元・40階化（v1.5）

- **ステージを元の内容へ戻しました**。v1.4で到達性のために緩めていた生成（中盤の穴を真裏付近に、危険帯の上限を4に）を元に戻しています。スワイプ速度ゲイン（v1.4）で遠い穴も素早いフリックで届くため、元の難度でもクリア可能です。
- **クリア目標を30階→40階に拡張**。生成ルール（シード／序盤±1・中盤真裏／穴の隣の危険／20階以降は穴1マス）はそのまま40階へ適用。クリアボーナス `+3000` は据え置き。
- その他（ゲーム内時間・スワイプ速度ゲイン・画面固定/誤バック防止・危険床赤・ファイア6階など）は維持。

## 画面フィット・スワイプ速度・誤バック防止（v1.4）

- **スワイプ速度で回転量が変わる**: 速く動かすほど大きく回ります（`gain = 1 + min(SWIPE_GAIN_MAX, |dx|×SWIPE_GAIN_K)`）。`dt` 割り算を使わないのでフレームレートで暴れず、遠い穴も素早いフリックで届きます。
- **深層の到達可能性を改善**: 中盤（11階〜）の穴を「真裏」から2〜3ブロックの中程度ズレに緩和し、危険帯の上限を3に。深層が一面赤で着地不能になり詰む問題を解消（全階で穴中央は必ず通過可能）。
- **プレイ中は画面を固定＝誤ブラウザバック防止**: プレイ中のみページを `position:fixed` で完全固定し全幅化、`touchmove` を抑止。画面の揺れや左端スワイプでの戻る誤操作を防ぎます。
- 詳細・検収は `docs/specification.md` の「v1.4」節を参照。

## 安定化・ステージ生成アップデート（v1.3）

- **タブ切替でコンボが消えない**: 時間管理を実時間（`performance.now()`）からゲーム内時間（`inGameTime`、ループ内で加算）へ置換。バックグラウンドで描画が止まっても進行・コンボが不自然に失われません。
- **スワイプが端末で安定（指に吸い付く）**: フレーム間 `dt` での割り算を廃止し、**指の移動ピクセル×固定係数**を回転へ直接反映。どの端末・フレームレートでも同じ指移動量＝同じ回転量（離した後だけ軽い慣性）。
- **ステージのプロシージャル生成**: プレイごとのシード＋階層ルールで毎回違うレイアウトに。
  - 1〜10階は穴を直前の±1ブロックに置いて連続落下を誘発（操作学習）、11階以降は真裏付近へ大きくズラして素早い回転を要求。
  - 危険帯は穴のすぐ隣などミスしやすい箇所へ確定配置（必ず通過できる肩は残す）。
  - 穴幅は段階縮小、**20階以降は必ず1マス**で終盤の緊張感。
- 詳細・検収は `docs/specification.md` の「v1.3」節を参照。

## 操作感・接触感・立体感アップデート（v1.1）

ゲーム内容を変えずに、Helix Jump系の手触りへ近づける調整を行いました。Codex実装（PR #10）と本実装を統合し、慣性・通過/失敗演出はCodex実装を採用、通過判定（依頼書準拠の球幅）と疑似3D奥行きを上乗せしています（詳細・検収は `docs/specification.md` の「v1.1」節）。

- **回転の重さ（慣性）**: 指の速度に追従して角速度（rad/秒）が乗り、離すと少し慣性で回って止まります。`MAX_ANG_V/ANG_DRAG` で調整可能。秒基準でfps差に強い。
- **球の幅を考慮した接触判定**: `collisionAt()` が正面の中心・左右端の3点を見て、**中心＋左右すべてが穴のときだけ通過**／端が床なら跳ね返り／危険帯に少しでも触れたら失敗。見た目と判定のズレを低減。
- **床通過の演出**: 通過の瞬間に球が縦へ伸び、穴の縁が一瞬光り、カメラ追従が速まって落下感を強調。
- **失敗理由の明示**: 危険帯に当たったくさびを赤くハイライトしてから結果へ。理由は従来どおり「危険床接触」。
- **疑似3Dの奥行き**: Canvas 2Dのまま、各スライスの手前を明るく・奥を暗くして立体感を強化（危険帯の視認性は維持）。
- `?debug` 付きURLでは接触判定（結果・中心/左右くさび・角度・span）を画面下部に表示します。

## Helix品質アップデート（2.1）

参照ゲーム（Helix Jump）とのビジュアル・ゲーム性の差異を役割分担で洗い出し、出荷品質へ引き上げました。

- **中実スライス床＋疑似3D**: 中空リング描画を廃止し、各床を中実のパイ状スライス（8分割）＋側面の厚みで描画。中央支柱と奥→手前の描画順で立体的な螺旋の塔に。
- **ファイアボール🔥**: 3連続落下で太陽が発火し、一定時間すべてのスライス（赤い危険帯も）を破壊して突き抜ける貫通アクション。コンボの報酬が「見える力」になりました（破壊ごとに加点）。
- **ゾーン配色**: 6階ごとに配色パレットを変え、背景を連続補間。危険帯は安全帯と高コントラストな色で警告性を確保。
- **ボール刷新と演出**: 陰影付きの球＋接地影＋着地スカッシュ＋着地スプラッシュ痕＋画面シェイク＋触覚振動。回転光線は廃止して上質に。
- 差異分析・採否・実装・検証は `docs/helix_fidelity_plan.md` を参照してください。

## 物理演算リニューアル（2.0）

iPhone SEで「太陽が全く動かない／塔の構造が読めない」という致命的課題を解消するため、コア物理と描画モデルを刷新しました。

- **物理演算**: 重力加速・速度積分・反発（restitution）を導入。固定タイムステップ（1/120秒）＋アキュムレータで、低/可変フレームレートのiOSでも当たり判定が破綻しません。
- **描画モデル刷新**: 同心円コイル状の旧描画を廃止し、遠近の楕円リングを縦に積層＋中央支柱で「塔」を明示。現在階の正面に状態色リング（緑＝穴 / 赤＝危険 / 白＝床）と階数ラベルを表示し、初見でも狙いどころが分かります。
- **演出（リアリティ）**: 太陽は自前描画（放射光の回転・着地スクワッシュ・落下ストレッチ）、影・残像・着地/取得/失敗の粒子・画面シェイク・触覚振動を追加。
- **iOS安定化**: `ResizeObserver` と `visualViewport`/`orientationchange` 購読、ループ内の再サイズガード、`min-height:100vh→100dvh` 二段指定で、Canvasが0/1pxになる不具合を防止。
- **読み込み**: 外部依存ゼロのHTML1枚（約31KB）。インラインSVGファビコンで余計なリクエスト無し。
- **維持した改善**: 名前入力のEnter送信、ランキング状態の `aria-live` 通知、シェアの共有/コピー/手動コピー多段フォールバック。
- 詳細なテストプレイ・役割分担レビュー・採否は `docs/physics_overhaul_plan.md` を参照してください。
